# KubeArmor Community

KubeArmor is a community-governed open source project and a [Cloud Native Computing Foundation (CNCF) Sandbox project](https://www.cncf.io/projects/kubearmor/). The project is developed in the open and welcomes contributions from individuals and organizations regardless of affiliation.

## Governance

KubeArmor is run by its Maintainers, with public processes for joining the project, making decisions, handling security, and shipping releases.

The source of truth for these processes lives in the main repository. Please refer to:

- **[Governance](https://github.com/kubearmor/KubeArmor/blob/main/GOVERNANCE.md)** — roles, decision-making, vendor neutrality, sub-teams, voting.
- **[Maintainers](https://github.com/kubearmor/KubeArmor/blob/main/MAINTAINERS.md)** — current Maintainers, Reviewers, and Emeritus Maintainers, with affiliations.
- **[Code of Conduct](https://github.com/kubearmor/KubeArmor/blob/main/CODE_OF_CONDUCT.md)** — KubeArmor follows the [CNCF Code of Conduct](https://github.com/cncf/foundation/blob/main/code-of-conduct.md).
- **[Release Process](https://github.com/kubearmor/KubeArmor/blob/main/RELEASES.md)** — cadence, release candidates, release manager, support window.
- **[Security Policy](https://github.com/kubearmor/KubeArmor/blob/main/SECURITY.md)** — how to report a vulnerability.

## Become a contributor

You don't need to be a coder to contribute. The project benefits from documentation, policy templates, integration work, talks, blog posts, and community management.

Start here:

- **[Contributing Guide](https://github.com/kubearmor/KubeArmor/blob/main/CONTRIBUTING.md)** — what kinds of contributions we welcome and how to start.
- **[Good first issues](https://github.com/kubearmor/KubeArmor/issues?q=is%3Aissue+is%3Aopen+label%3A%22good+first+issue%22)** — small, well-scoped issues if you're new.
- **[Help-wanted issues](https://github.com/kubearmor/KubeArmor/issues?q=is%3Aissue+is%3Aopen+label%3A%22help+wanted%22)** — issues where the community would value extra hands.
- **[Backlog issues](https://github.com/kubearmor/KubeArmor/issues?q=is%3Aissue+is%3Aopen+label%3Abacklog)** — larger work items waiting for an owner.

If you want to grow into a Maintainer role, the [Roles section of GOVERNANCE.md](https://github.com/kubearmor/KubeArmor/blob/main/GOVERNANCE.md#roles) lays out the tiers and what each one requires.

## Mentorship

KubeArmor participates in CNCF and Linux Foundation mentorship programs:

- **[Google Summer of Code (GSoC)](https://summerofcode.withgoogle.com/)** — KubeArmor maintainers mentor accepted students on scoped projects each summer.
- **[LFX Mentorship](https://lfx.linuxfoundation.org/tools/mentorship/)** — paid mentorships under CNCF/LF, typically a few cycles per year.

Active and upcoming mentorship projects are posted on the [LFX Mentorship portal](https://lfx.linuxfoundation.org/tools/mentorship/) when applications are open.

## Project repositories

## Reporting a Code of Conduct violation

If you experience or witness behavior that violates the [CNCF Code of Conduct](https://github.com/cncf/foundation/blob/main/code-of-conduct.md), you can:

- Contact the project Maintainers via the channels listed in [MAINTAINERS.md](https://github.com/kubearmor/KubeArmor/blob/main/MAINTAINERS.md), or
- Email the CNCF directly at [conduct@cncf.io](mailto:conduct@cncf.io) — especially if the report concerns project leadership.

See the project's [CODE_OF_CONDUCT.md](https://github.com/kubearmor/KubeArmor/blob/main/CODE_OF_CONDUCT.md) for the full reporting path.

## Reporting a security vulnerability

Please **do not** open a public GitHub issue for security bugs. Follow the process in [SECURITY.md](https://github.com/kubearmor/KubeArmor/blob/main/SECURITY.md). The KubeArmor Security Response process is also described in the [Sub-teams section of GOVERNANCE.md](https://github.com/kubearmor/KubeArmor/blob/main/GOVERNANCE.md#security-response-committee-src).

## CNCF affiliation

KubeArmor is a [Sandbox project](https://www.cncf.io/projects/kubearmor/) of the Cloud Native Computing Foundation. CNCF stewards a neutral home for the project and provides infrastructure, marketing, and legal support.

The project's CNCF status, marketing assets, and TAG interactions are governed by the rules in [GOVERNANCE.md](https://github.com/kubearmor/KubeArmor/blob/main/GOVERNANCE.md#cncf-requests-and-integrations).
